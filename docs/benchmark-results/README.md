Important limitation:

- the benchmark harness itself now runs correctly and no longer hangs the whole suite
- however, the default long-duration `quick` / `full` runs still expose a library-side keep-alive stability issue in `http_get_route_param/light` and `http_session_counter/light`
- because that is a product bug rather than a harness bug, this directory currently ships a clearly labeled **smoke snapshot**, not a full official baseline

Once the library issue is fixed, this directory should be updated with a full multi-round snapshot generated by:

```bash
./build-bench/benchmarks/bsrvcore_http_benchmark \
  --profile full \
  --output-json docs/benchmark-results/<date>-<machine>-full.json
```
